React笔记1——React介绍

React是什么

  • React 是一个用于构建用户界面的 JavaScript 库
  • React源于Facebook的内部项目，当时该公司因为对市场上的所有JavaScript MVC框架都不太满意，就自己写了一套框架，用来Instagram网站
  • React的设计思想独特，性能出众，代码逻辑简单，逐渐称为Web开发的主流工具

组件化思想

  • 什么是模块化：从代码的角度分析，把一些可复用的代码，抽离为单个模块，便于项目的维护和开发
  • 什么是组件化：从UI界面的角度分析，把一些可复用的UI元素，稠李为单个组件，便于项目的维护和开发
  • 组件化的好处：随着数目规模最大，手里的组件越来越多。那么可以很方便地把现有的组件，拼接成一个完整的页面

为什么要学习React

  • React的设计优秀，一切基于JS并且实现了组件化开发的思想
  • 开发团队一直在不断更新该框架
  • 社区强大，很多问题都有相应的解决方案
  • 很多大型企业都在应用React这套框架

核心概念

  • Hello World(react的简单应用)
  • JSX
  • 元素渲染
  • 组件&Props
  • State&生命周期
  • 事件处理
  • 条件渲染
  • 列表&Key
  • 表单
  • 状态提升
  • 组合vs继承
  • React哲学
  • 常见API
  • Hook

  • 在cmd环境中输入以下命令
  • Create React App 不会处理后端逻辑或操纵数据库；它只是创建一个前端构建流水线（build pipeline），所以你可以使用它来配合任何你想使用的后端。它在内部使用 Babel 和 webpack.
  • 部署生产环境：执行npm run build会在build文件夹生成应用的优化版本

JavaScript工具链

  • 一个package管理器，比如yarn或npm，可以充分利用第三方package生态系统，轻松安装或更新它们
  • 一个打包器：比如webpack或parcel。可以编写模块化代码，并且组合成小的package，优化加载时间
  • 一个编译器：比如Babel，可以编写最新版本的JS代码，并且依然能够在旧版浏览器工作
