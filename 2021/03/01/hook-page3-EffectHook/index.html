ReactHook笔记3——Effect Hook

useEffect是什么

  • Effect Hook 可以让你在函数组件中执行副作用操作
  • 常见的副作用操作有获取数据，设置订阅，手动更改React组件的DOM等
  • 官网解释道，useEffect可以看作componentDidMount，componentDidUpdate 和 componentWillUnmount 这三个函数的组合
  • 常见的副作用操作分为两种：需要清除的和不需要清除的

无需清除的effect

  • 有时只是想在更新DOM之后运行一些额外的代码，例如发送网络请求、手动变更DOM、记录日志等，都是常见的无需清除的操作，执行完之后就可以忽略它们了
  • 示例如下代码所示
  • useEffect做了什么：这个Hook可以告诉react组件在渲染后需要执行的操作
  • 为什么在组件内部调用 useEffect：将 useEffect 放在组件内部让我们可以在 effect 中直接访问 count state 变量（或其他 props）。Hook 使用了 JavaScript 的闭包机制，而不用在 JavaScript 已经提供了解决方案的情况下，还引入特定的 React API。
  • useEffect什么时候执行：它在每次渲染后和每次更新后都会执行
  • React 何时清除 effect： React 会在组件卸载的时候执行清除操作。

useEffect的依赖项

  • 某些情况下，每次渲染后都执行清理或者执行 effect 可能会导致性能问题。
