CSS学习笔记-css的加载方式

发表于2021-07-16|更新于2021-07-16|CSS
|阅读量:

css的加载方式

  • css的加载方式一共有四种

    1. 行内样式
    • 直接对HTML标签引入style=””设置样式；
    • 优缺点：会使得html不干净，代码冗余不利于维护。
    1. 内嵌样式–内部样式表
    • 在head标签之间用**标签**声明
    • 优缺点：页面使用公共CSS代码，也是每个页面都要定义的，如果一个网站有很多页面，每个文件都会变大，后期维护难度也大，如果文件很少，CSS代码也不多，这种样式还是很不错的。
    1. 链入外部样式表—在head部分加入
    • 链接样式使用最多，推荐使用的一种方法，在标签之间用<link …../>引入css文件
    1. 导入外部样式表—在head部分加入
    • 导入样式和链接样式比较相似，采用**@import样式**导入CSS样式表，在HTML初始化时，会被导入到HTML或者CSS文件中，成为文件的一部分，类似第二种内嵌样式。

四种加载方式的优先级

1. 就近原则

2. 理论上：行内>内嵌>链接>导入

3. 实际上：内嵌、链接、导入在同一个文件头部，谁离相应的代码近，谁的优先级高

网络堵塞会影响css的加载吗

* 结论：

    1. 网络堵塞，css加载不会影响DOM树的解析
    2. 网络堵塞，css加载会阻塞DOM树的渲染（这个机制是为了减少回流重绘。因为如果正常渲染，那么加载完之后可能要重新回流重绘）
    3. css加载会阻塞后面js语句的执行

* 提高css加载速度的方法：

    1. 使用CDN（内容分发网络）
    2. 对css进行压缩
    3. 合理使用缓存（设置cache-control，expires以及E-tag）
    4. 减少http请求次数，合并多个css文件
