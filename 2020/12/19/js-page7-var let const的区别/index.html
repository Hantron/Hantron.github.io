JavaScript学习笔记-var let const的区别

发表于2020-12-19|更新于2021-01-09|JavaScript
|阅读量:

var let const的区别

变量

ES的变量时松散类型的，即变量可以用于保存任何类型的数据。每个变量是一个用于保存任意值的命名占位符。

有三个关键字可以声明变量：var、let、const，其中let和const是 ES6才开始使用的。

var关键字

  1. var声明作用域：var定义的变量会称为包含它的函数function的局部变量。比如说在一个函数内部用var定义变量，意味着该变量会在函数退出时被销毁
  2. var声明提升：之前有提到过，在这里不再叙述
  3. 全局声明：var在全局作用域中声明的变量会成为window对象的属性
  4. 可以重复定义，可以修改值，支持for循环

let关键字

  1. let声明作用域：let声明的范围是块级作用域，而var声明的范围是函数作用域
  2. let不会声明提升，若在let声明某变量前调用该变量，那么会进入暂时性死区，抛出错误ReferenceError
  3. 全局声明：let在全局作用域中声明的变量不会成为window对象的属性，而var会。
  4. for循环：使用var时，for循环定义的迭代变量会渗透到循环体外部。使用let则不会出现这个问题
  5. let不能重复定义，可以修改值。

const关键字

  1. const的行为与let基本相同，声明范围也是块级作用域，但是唯一重要的区别的是用const声明变量时必须同时初始化变量，否则会报错。
  2. 但是，const声明的限制只是限制它指向的变量引用。也就是说，如果const变量引用的是一个对象或者数组，那么修改这个对象或数组内部的内容，不会违反const的限制。
  3. 由于const声明的变量不可改，因此const不支持for循环

总结

由于ES6增加了let和const的关键字，使得这门语言更精确地声明作用域和以及语义得到了更好的支持。行为怪异的var会经常造成各种问题，这两个新关键字的出现，有助于代码质量的提高。
