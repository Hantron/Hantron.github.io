React笔记4——生命周期

生命周期方法

  • 示例，以下为一个获取时间的时钟class组件

  • 将生命周期方法添加到class组件中

    • 当Clock组件第一次渲染到DOM中的时候，就为其设置一个计时器。此时称为**”挂载(Mount)”**
    • 当Clock组件被删除的时候，应该清除计时器。此时称为”卸载(unmount)”
    • 我们可以声明方法componentDidMount()和componentWillUnmount()在组件挂载或卸载时执行（即生命周期方法）

  • 回顾以下发生了什么以及方法的调用顺序
    • 被传给ReactDOM.render()的时候，React会调用Clock组件的构造函数。
    • 之后会调用组件的render()方法.用于确定React在页面上的展示方式，然后React更新DOM来匹配Clock渲染的输出。
    • Clock的输出被插入到DOM后，React会调用ComponentDidMount() 生命周期方法，即设置一个定时器每秒调用依次组件的tick()方法
    • 浏览器每秒调用一次tick()方法，tick()内部的setState()进行state的改变和UI更新，state改变后，会重新调用render()方法，然后更新DOM
    • 一旦Clock组件从DOM中移除，React就会调用componentWillUnmount() 生命周期方法

组件的生命周期

  • 每个组件都包含”生命周期方法”，可以查阅生命周期图谱来使用
  • 记住下面加粗的常见方法即可

挂载

  • 当组件实例被创建并且插入到DOM中时，生命周期的调用顺序如下：
    • constructor()
    • static getDerivedStateFromProps()
    • render()
    • componentDidMount()

更新

  • 当组件的props或state发生变化时，会触发更新
  • 组件更新的生命周期调用顺序如下：
    • static getDerivedStateFromProps()
    • shouldComponentUpdate()
    • render()
    • getSnapshotBeforeUpdate()
    • componentDidUpdate()
